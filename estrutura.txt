## 📁 Estrutura do Projeto `post-install-linux`

Este projeto organiza scripts e dados para automatizar a instalação e configuração de sistemas Linux após a instalação. Ele é modular, extensível e preparado tanto para execução via terminal quanto para futura interface gráfica com GTK4.

```
post-install-linux/
│
├── data/                 
│   ├── arch.json         # Lista de pacotes específicos para Arch Linux
│   ├── debian.json       # Lista de pacotes para sistemas baseados em Debian/Ubuntu
│   ├── fedora.json       # Lista de pacotes para Fedora
│   └── common.json       # Pacotes comuns a todas as distribuições
│
├── resources/            
│                         # Arquivos estáticos: ícones, imagens, temas, arquivos .desktop etc.
│
├── src/
│   ├── shell/            
│   │   ├── configs.sh           # Configurações do sistema (ex: aliases, locale, etc)
│   │   ├── install_packages.sh  # Lógica de instalação de pacotes com base nos arquivos JSON
│   │   ├── theme.sh             # Aplicação de temas e aparência
│   │   └── utils.sh             # Funções auxiliares usadas por outros scripts shell
│   │
│   └── python/           
│       └── list_packages.py     # Script em Python para listar pacotes disponíveis por distro
│
├── gui/
│   └── main_window.py     # Estrutura base da interface gráfica com GTK 4 (ainda em desenvolvimento)
│
├── main.sh               # Script principal para execução via terminal (CLI)
├── postinstall.py        # Entrada principal do sistema em Python. Pode detectar e rodar GUI ou CLI.
├── README.md             # Descrição geral do projeto, instalação e uso
├── LICENSE.txt           # Licença do projeto
├── .gitignore            # Arquivos e pastas a serem ignorados pelo Git
└── estrutura.txt         # (Opcional) Esboço da estrutura do projeto. Pode ser movido para docs/
```

---

### ✅ Convenções adotadas

* **Separação de responsabilidades**: dados, scripts bash, scripts Python e interface gráfica estão organizados em pastas distintas.
* **Modularidade**: scripts pequenos e específicos, facilitando manutenção e testes.
* **Pronto para CLI e GUI**: `main.sh` para terminal e `postinstall.py` para integração com GTK4.
* **Multi-distro**: suporte inicial para Arch, Debian e Fedora.
